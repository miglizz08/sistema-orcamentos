FRAMEWORKS: BOOTSTRAP

adiconar o atributo class dentro das tags:

1. bg-dark: Pinta o fundo de escuro (background dark).
2. text-white: Pinta a fonte de branco.
2. text-center: Centraliza o texto.
4. p-5: adiciona um preenchimento (padding) grande por dentro, para a caixa não ficar espremida.

<header class=" bg-dark text-white text-center p-5">
	<h1>Meu nome</h1>
	<p>Construo sistemas rápidos para o seu negócio.</p>
</header>

No Bootstrap, nós temos um padrão de cores que usa nomes lógicos de intenção em vez de códigos hexadecimais complexos:
1. primary
2. success
3. danger
4. dark

Para mudar a cor de um texto, usamos o prefixo text- seguido do nome da cor (exemplo: text-success).

usamos o prefixo mb- (que significa Margin Bottom, ou margem inferior) seguido de um número de 1 a 5 (exemplo: mb-3).

O Bootstrap tem um componente visual chamado List Group que transforma essas bolinhas simples em blocos elegantes, parecidos com menus de aplicativos. Para que essa mágica aconteça, precisamos formar um "par" de classes:

A caixa principal (<ul>) precisa receber a classe list-group.

Cada item individual (<li>) precisa receber a classe list-group-item.

<ul class="list-group">
    <li class="list-group-item">Sistemas rápidos e otimizados</li>
    <li class="list-group-item">Foco na resolução do problema do cliente</li>
    <li class="list-group-item">Código seguro e organizado</li>
</ul>

Um link de texto normal (aquele azul sublinhado) pode passar despercebido. Mas o Bootstrap tem um "truque" incrível: ele pode transformar qualquer link em um botão chamativo e clicável, perfeito para a tela do celular.

Para fazer isso, nós usamos duas classes em conjunto, parecidas com a lógica da lista:

btn: Dá o formato, as bordas e o espaçamento de um botão.

btn-primary: Pinta o botão com a cor principal de destaque (azul).

Area dos Meus Projetos:

Dividindo as 12 colunas totais por 3, chegamos a 4. Então, a classe correta para cada projeto ocupar um terço da tela é a col-4.

No Bootstrap, para que essa mágica de posicionamento funcione, nós precisamos de duas peças principais trabalhando juntas, substituindo aquelas antigas tags de lista (<ul> e <li>):

A linha (onde os elementos vão ficar lado a lado), que criamos usando <div class="row">.

A coluna (o espaço de cada elemento), que acabamos de descobrir que será <div class="col-4">.

<div class="row">
	<div class="col-4">
		</div>
</div>



Colocando todas as caixas (col-4) dentro da mesma "prateleira" (row), nós garantimos que elas fiquem perfeitamente alinhadas lado a lado.

Como a soma das três colunas dá exatamente as 12 colunas matemáticas do Bootstrap (4 + 4 + 4 = 12), elas vão preencher o espaço da linha inteira de forma totalmente simétrica.

O seu código HTML com essa estrutura completa ficou assim:

HTML
<div class="row">
    <div class="col-4">
        </div>
    
    <div class="col-4">
        </div>
    
    <div class="col-4">
        </div>
</div>

O Raio-X do Card:
Aqui está o que cada uma dessas novas classes do Bootstrap está fazendo no seu design:

class="card": É a caixa principal. Ela cria automaticamente uma borda cinza bem suave e cantos arredondados, dando aquele aspecto de cartão físico.

class="card-img-top": Diz para a imagem se adaptar perfeitamente ao topo do cartão, arredondando apenas os cantos superiores para acompanhar o design. (No lugar daquele link via.placeholder, o senhor colocará o caminho para o "print" da sua tela real).

class="card-body": Cria um espaçamento (padding) em volta dos textos, para que as palavras não fiquem coladas na borda do cartão.

class="btn btn-primary": Transforma um link comum (<a>) em um botão azul, bonito e clicável. O segredo de tudo está no atributo href="...": é ali dentro que o senhor vai 


Um pequeno ajuste:

**O Ajuste para Celulares (Responsividade)**

Lembra que conversamos sobre como o site ficaria no smartphone? Se o senhor usar apenas a classe `col-4`, o Bootstrap vai obedecer cegamente e espremer os 3 projetos lado a lado na tela estreita de um celular. O texto vai ficar ilegível e a imagem minúscula.

Para resolver isso, nós fazemos um ajuste minúsculo, mas poderoso. Em vez de escrever apenas `<div class="col-4">`, o senhor vai usar:

`<div class="col-12 col-md-4">`

Aqui está a tradução do que o senhor está dizendo para o navegador com esse comando:
* **`col-12`**: "Quando a tela for pequena (celular), quero que este cartão ocupe as 12 colunas da tela". Isso faz os projetos ficarem bonitos, grandes e empilhados um embaixo do outro.
* **`col-md-4`**: "Quando a tela for Média (`md`) ou maior (computador), volte a usar apenas 4 colunas". Isso faz os 3 projetos ficarem lado a lado novamente.
